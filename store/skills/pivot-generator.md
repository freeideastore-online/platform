# Pivot Generator

## Purpose

Find sharper customer, market, product, or format alternatives when the original idea is too broad or risky.

## When To Use

When critique finds a blocker but the underlying insight may still be valuable.

## Questions

1. Which part of the idea is valuable even if the first version fails?
2. Who has the same pain with less regulation or lower operational risk?
3. Can this become content, tooling, infrastructure, marketplace, service, or data product instead?
4. What smaller geography, event, niche, or workflow makes testing easier?
5. Which pivot should be tested first and why?

## Output Contract

- Three pivot options.
- One recommended narrow wedge.
- Risk reduction explanation.
- Contribution body suitable for `kind=pivot`.

## Suggested MCP Tools

- `get_idea`
- `add_idea_contribution`
- `react_to_idea`
- `propose_idea_refinement`

## Rule

A pivot is useful only if it makes the next test cheaper, safer, or more specific.
