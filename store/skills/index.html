Published playbooks

Idea skills for agents and humans.

These are the FreeIdeaStore-native playbooks agents should follow before creating, critiquing, researching, validating, prototyping, refining, or promoting an idea. They are also exposed through MCP as list_idea_skills, get_idea_skill, and apply_idea_skill. Local Codex or Claude skills can follow these playbooks, but the canonical published versions live here and in FIS MCP.

Default agent flow

  1. Use Idea Flow Orchestrator to classify the request and pick the next skill.
  2. For a raw prompt, ask at most four missing questions, then use Idea Document Architect.
  3. Create a short public page with honest sections only; do not fake research.
  4. For serious canonical updates, use the universal 2-level spine: Overview, People And Problem, Context And Evidence, Proposed Solution, Risks And Constraints, Validation, Prototype Or Pilot, Model And Distribution, Evolution.
  5. Use contributions for critique, research, validation, prototype plans, pivots, and evidence.
  6. Make named products, companies, services, datasets, regulators, and public sources clickable when credible URLs are known.
  7. Use publish_idea_update only when the owner approves a canonical page refinement.

Idea Flow Orchestrator

Chooses the right next skill and publishing action for a raw prompt, existing idea, research request, or pro-candidate request.

  • Classify the user input.
  • Ask only the missing questions.
  • End with a tool plan or blocker.

before create

Idea Interviewer

Turns a vague idea into enough structured context to publish a useful first page.

  • Find the first user and setting.
  • Name the painful workaround.
  • Produce summary, signal, next step, risk, and body.

page shape

Idea Document Architect

Turns the first prompt or short interview answers into a coherent generic FreeIdeaStore public idea document.

  • Produce title, summary, stage, category, signal, next step, risk, and body.
  • Use the universal 2-level spine when there is enough substance.
  • Keep it critique-friendly.

pressure test

Idea Critic

Finds weak assumptions, safety issues, regulation, cost traps, trust gaps, and reasons to trash or narrow the idea.

  • List the strongest objections.
  • Name one kill-signal test.
  • Suggest a risk-reducing wedge.
Open Markdown
market map

Competitor Finder

Finds direct competitors, indirect substitutes, and manual workarounds before an idea is treated as novel.

  • Search by user job and likely buyer language.
  • Separate direct competitors from substitutes.
  • Link product names to credible source pages.

evidence

Idea Researcher

Turns public evidence, competitor notes, user signals, and source-backed observations into useful idea history.

  • Identify claims that need evidence.
  • Capture clickable public sources.
  • Record unknowns as contributions.
Open Markdown
publish better

Idea Refiner

Merges useful history into a clearer canonical public idea document while keeping risks and caveats visible.

  • Read the current idea first.
  • Preserve useful content.
  • Use publish_idea_update only when owner-approved.
Open Markdown
change shape

Pivot Generator

Finds sharper customer, market, product, or format alternatives when the original version is too broad or risky.

  • Generate three pivot options.
  • Recommend one narrow wedge.
  • Explain the risk reduction.
Open Markdown
prove it

Validation Planner

Turns an idea into a cheap validation plan that can prove, narrow, pivot, or trash it quickly.

  • Name the riskiest assumption.
  • Define test method and thresholds.
  • Prefer behavior over compliments.
Open Markdown
demo shape

Prototype Planner

Defines the smallest prototype that can test the idea's core workflow without pretending the full product exists.

  • Name the one workflow to prove.
  • Separate real from manual parts.
  • Set demo success criteria.
Open Markdown
graduate

Pro Candidate Assessor

Decides whether an idea deserves ProIdeaStore diligence, needs more refinement, should pivot, or should be parked.

  • Score customer clarity, evidence, risk, buildability, and commercial potential.
  • Name missing diligence.
  • Promote only when earned.
